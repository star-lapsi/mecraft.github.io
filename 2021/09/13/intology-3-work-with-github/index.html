Intology项目(3) - Github使用笔记

发表于 2021-09-13 更新于 2021-09-17 分类于 Intology项目 阅读次数：

在线托管

以前的代码托管基本是各个团队自己内部搞一个局域网，简单的通过共享文件夹，复杂的搞一个SVN之类的服务器来实现远程仓库的作用。但是对于需要集中全世界力量的开源社区来说这么做就不太方便了，因此一个比较体系化的在线代码托管平台就成了需求所在，github和gitee就是这类的产品。

需要做什么

不论是github还是gitee，目的是一致的，就是申请一个在线服务器空间，然后明确空间的用途，最后将代码通过工具传输到该平台上，实现本地和在线的同步。因此我们需要做的事情很明确：

  1. 找到一个平台。
  2. 注册并验证账号。
  3. 创建远程仓库。
  4. 远程仓库初始化（平台自动进行，用户在创建时需要做些配置）。
  5. 安装git或类似软件。
  6. 确定远程仓库登录方式（分为密码登录及密钥登录，建议选择后者）。
  7. 创建密钥并配置到仓库（密码登录不需要）。
  8. 执行提交。

    一些细节说明

    流程是明确的，实际操作中，依据提示选择默认即可。这里主要提一下一些细节问题：
  9. 平台选择：github或gitee都是可用的，gitee是国内平台，国内访问会更加方便一些，而且界面基本是全中文，英文不好的情况下比较容易接受。
  10. 仓库本质：创建仓库，本质上类似于服务器提供了一个远程路径，假设我的github用户名是aaa，我创建的仓库是hello，那么这个仓库的地址就是：<github.com/aaa/hello>。有些平台比如github会提供仓库的第三方访问页，这主要是应对需要托管网站代码这类代码的需要，配置好了仓库对应的网址之后，可以通过浏览器访问。
  11. 所谓密钥：所谓密钥，分为私钥和公钥，公钥是公开的，相当于锁头；私钥是个人保留，相当于钥匙，我们通过Git和平台交互数据时，平台用公钥加密数据，而我们则通过私钥进行解密。锁头可以交给任何人，但是私钥必须自己保留，否则泄露出去会出大问题。由于一般代码在仓库之间的交互是通过SSH协议进行，所以如果要通过密钥登录，可以用SSH产生对应的密钥：

    ssh-keygen -t rsa

产生的密钥一般都在用户文件夹底下的.ssh文件夹内，分为id_rsa.pub（公钥）和id_rsa（私钥），后者我们自己保留，前者我们需要配置到托管平台上（一般都有SSH密钥设置，需要将公钥内容复制或者上传）。完成之后，就可以进行提交验证，比如github为：

ssh -T git@github.com
