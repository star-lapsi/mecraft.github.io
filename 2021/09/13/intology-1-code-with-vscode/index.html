创作的过程

从最简单的理解来说，码代码就像是在写一个流程，我用某种语言写下：

但是我所用的语言有一个大问题：电脑看不懂！这个时候需要一个特殊的程序把它进一步翻译成：

所以我们往往需要两个最基础的工具：编辑器（写流程）和编译器/解释器（翻译），有了这两个工具你就可以进行创作了。当然了，这里还需要另一个工具，为了方便排查问题，我们还需要一个调试的工具，调试工具一般会作为编译器的补充存在。

试试VSCode

不想废话太多，就拣最简单粗暴的说为什么用VS Code：

  1. 有界面，说明可以用鼠标做很多操作，免掉很多指令的学习成本。
  2. 有开源，别的不说至少免费。
  3. 有品牌，来自企业端最大的好处就是安装了就能码字，品牌效应下也有足够的社区效应，不怕找不到合适的解决方案。
  4. 有插件，插件能做的事情很多，除了美化界面以方便码字外，插件也可以减少一些操作，比如直接调试、运行，不用跑来跑去，而且插件可以自由组合，不用安装一堆东西（好过装一个比较死板的IDE）。
