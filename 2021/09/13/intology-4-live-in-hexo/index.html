Intology项目(4) - Hexo搭建笔记

发表于 2021-09-13 更新于 2021-10-06 分类于 Intology项目 阅读次数：

目的在于内容

如果你要搭建个人博客，那么你需要做的准备工作看上去非常多：购买服务器和域名，在服务器上编写网站框架及代码，配置访问的协议细节……如果一个人要从头去啃HTML等网页设计相关的技术，那么网站的建立估计就得“无限推迟”。而Hexo是一个非常简单的框架，它内置了一些基本的模板，你可以在其基础上进行自定义，这个框架就可以帮你生成你所需要的页面效果。这样一来，你就省去了自己费尽周折去设计网页元素的问题，转而可以把更多精力用在内容的完善上。

安装过程

安装Hexo并不是通过下载exe进行，而是通过包管理器，借助指令进行安装。这可能是因为这个框架的目的是为了在特定的服务器运行的，大多数时候服务器的维护人员是没有GUI界面（Linux服务器）去折腾的，包管理器+指令集则是一种更加高效的方案。因此，安装次序是这样的：

安装Node.js的工作，在windows下只需要到官网下载安装即可；在苹果系统以及部分Linux系统中，系统提供了包管理器来进行下载，这部分具体内容可以直接参考Node.js官方主页的介绍，安装完成后需要验证Node.js以及npm包管理器是否正确安装：

node -v
npm -v

通过npm包管理器安装hexo时会有一个问题，那就是需要配置环境变量信息，并且需要在node.js的主文件夹下面创建文件夹node_cache以及node_global，前者是缓存区域，后者则是通过包管理器下载的软件的安装位置。创建文件夹之后，需要把这两个文件夹首先配置到npm内：

npm config set prefix “node_global完整路径”
npm config set cache “node_cache完整路径”

完成上述配置后，就需要在环境变量（windows、mac和linux配置方式各不相同）里面配置这些信息，其中主要包含：

  1. 系统变量新增“NODE_PATH”并配置“node_global/node_modules完整路径”。
  2. 用户变量已有的“PATH”内补充“node_global完整路径”。

完成环境变量配置后，可以下载一个webpack模块，并进入node_global/node_modules里面查看是否安装成功：

npm install webpack -g

测试通过后，使用下列语句安装hexo-cli模块，也就是我们所需要的hexo的框架：

npm install hexo-cli -g

通过npm管理器安装的插件，我们可以通过下列指令进行查看，由于git包含bash文件的功能，因此可以配合grep进行筛选：

npm list

基本操作

初始化博客：

hexo init

清除之前产生的缓存信息：

hexo clean

用Hexo生成博客页面：

hexo g #g或者generate

将博客页面发布到本地（可以打开localhost:4000进行查看）：

hexo s #s或者server

将博客页面发布到代码托管平台：

hexo d #d或者deploy

发布到服务器或github

github模式

我们首先可以在服务器上面创建一个项目，创建过程和其他项目相同，不过建议名字改为“用户名.github.io”的模式（与后面你要设定的blog网址同名）。
按照上述的基本操作，在本地创建一个blog项目文件夹并初始化博客，执行清除缓存、生成界面之后，发布到本地进行查看。修改本地Blog文件夹下面的_config.yml文件，填写下列内容：

deploy:
type: git
repository: https://github.com/xxx/xxx.github.io.git #仓库地址，可以从页面上复制
branch: master #视具体情况，也可以是main

输入下列代码安装hexo-deployer-git：

npm install hexo-deployer-git –sava # Git部署插件安装

安装完成后，执行hexo部署三部曲，即可输入网址xxx.github.io查看自己的blog（xxx是你前面设定的名称）：

hexo clean
hexo g
hexo d
